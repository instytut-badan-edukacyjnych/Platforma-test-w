Libraries and data processing

Libraries

Report templates are created using HTML, JS i CSS, just like typical web pages. The report works in the local space of the device. It allows to use hyperlink-based navigation and to utilize external resources bundled with the report.

The application neither provides any libraries, nor requires any to be used. The report authors are free to choose any tools and libraries they know or like to use. The only requirement is that all modules used by the report must be able to run without so-called "server-side". That is, all libraries must be JavaScript-based and designed to run in the browser. This means no possibility of using libraries based on Java, PHP or ASP.Net or others.

This example of a report already uses several JavaScript libraries. A D3 (http://d3js.org/) library is used for generating the graphs below, and jQuery (http://jquery.com/) library is used to simplify all operations on the data and page layout. Several customized JavaScript functions are also present that provide some simple grouping algorithms to make the graphs more interesting.

Data visualization

The application provides only raw data that were stored by the application during examinations. Any required reference data (norms, levels, calibration data, etc) can be simply bundled with the report as an external resource and can be later loaded by the report. The actual format of that additional data is not specified and report authors can choose any, provided that the JavaScript code will be able to read and process it.

Data provided by the application can be displayed in raw form, but usually it will need to be processed and transformed into some form required by a specific visualization library. The main set of the data the report is provided with contains a series of examination results. A simple analysis of these data allows to discover, for example, in what institutions and departments the actual examinations took place.
